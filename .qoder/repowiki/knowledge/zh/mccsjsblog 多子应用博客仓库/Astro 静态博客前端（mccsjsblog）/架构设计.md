采用 Astro 内容路由驱动：`src/pages/` 下按资源域划分目录（`posts/[slug]`、`categories/[slug]`、`tags/[slug]`、`archive/page/*`），配合 `rss.xml.ts` / `sitemap.xml.ts` / `atom.xml.ts` / `robots.txt.ts` 等动态路由产出 SEO 文件；`src/layouts/BaseLayout.astro` 作为唯一布局入口，在 SSR 阶段统一 fetch `/api/settings`、`/api/menus`、`/api/friends` 注入全局设置与导航，并通过 `<slot name="head">` 供子页面注入 JSON-LD 等 head 片段。

- 组件层 (`src/components/`)：纯展示型 Astro 组件（Header/Footer/Breadcrumb/FloatingBar/PostCard/TwikooComments 等），其中仅 CommentForm 使用 `.tsx` 以复用 React 生态。
- 样式层：Tailwind CSS v4 通过 `@tailwindcss/vite` 插件接入，全局样式集中在 `src/styles/global.css`，主题色通过 CSS 变量（如 `--card-bg`、`--fg-muted`）在 BaseLayout 中提供。
- 数据源：所有运行时数据来自后端 `http://localhost:4000/api/*`，`getStaticPaths` 在构建期拉取全部 post slug 生成静态路由，其余数据在页面渲染时同步 fetch。
- 客户端交互：`public/scripts/base-layout.js` 由 BaseLayout 引入，配合 `@swup/astro` 实现无刷新页面切换；文章页内嵌脚本负责代码块工具栏、Fancybox 图片画廊、阅读进度条与 Buttefly 风格目录联动，均通过 `astro:page-load` / `swup:page-view` 事件重入，避免重复绑定。
- 依赖方向：pages → layouts → components → utils/types，不反向依赖；Markdown 解析在服务端用 markdown-it + highlight.js 完成，图片链接经正则改写为 fancybox 可点击结构。
