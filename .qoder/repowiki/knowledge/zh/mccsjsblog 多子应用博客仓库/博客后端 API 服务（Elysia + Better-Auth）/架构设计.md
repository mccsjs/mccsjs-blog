单入口单体应用：`src/index.ts` 是唯一的进程入口，集中注册所有路由；`src/auth.ts` 通过 `better-auth` + `@better-auth/prisma-adapter` 挂载认证插件并以 Elysia macro 形式注入 `{ user, session }` 上下文；`src/db.ts` 暴露单例 PrismaClient。

分层结构：
- 中间件层：CORS（按 NODE_ENV 区分生产/开发白名单）、静态文件 `/uploads`、认证宏、全局错误处理（VALIDATION→422、NOT_FOUND→404、其余→500）。
- 业务路由层：按资源划分 `/api/posts|categories|tags|comments|menus|friends|admin/*|settings|upload`，全部使用 Zod schema 校验请求体，管理员接口统一以 `{ auth: true }` 守卫。
- 数据访问层：Prisma Client 直接调用，事务用于批量 settings upsert。
- 启动期初始化：默认菜单、聚合菜单、友链种子数据在进程启动后执行一次；定时任务通过 `setTimeout`+`setInterval` 每 12 小时对友链发起双通道测速（XXAPI + 直连 fetch）。

依赖方向：index → (auth, db)；auth → db；无反向依赖，保持扁平。
