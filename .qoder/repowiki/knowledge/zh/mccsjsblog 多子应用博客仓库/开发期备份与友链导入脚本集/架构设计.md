本模块为独立 CLI 工具集合，无共享库，每个文件自包含一个可执行脚本：
- `backup.js` / `restore.js`：基于 Node fs/path 的轻量备份-恢复对，以仓库根 `D:\codex\mccsjsblog` 和 `.backup/` 为约定目录，通过 `<原文件名>_<ISO时间戳><扩展名>` 命名保留历史版本，并追加写入 `.backup/backup.log`；`restore.js` 支持 `list <路径>` 子命令及交互模式读取最近 20 条日志。
- `backup-watch.js`：在 `backup.js` 基础上引入 chokidar 监听 `**/*`，内置 `IGNORE_DIRS`/`IGNORE_EXTS` 白名单过滤，首次运行自动 `npm install chokidar --save-dev`，并对 change 事件做 800ms 防抖，同时提供 `once` 参数触发单次全量扫描（递归 walk）。
- `import-friends.mjs`：ESM 脚本，使用 `@prisma/client` 直接读写 `friendType`/`friend` 表，将 `D:/blog/source/_data/link.yml` 中的分组与链接写入数据库，缺失截图时回退到 WordPress mshots 服务生成预览图。
依赖方向单向：所有脚本仅依赖 Node 标准库或显式声明的外部包（chokidar、yaml、@prisma/client），不互相引用。
