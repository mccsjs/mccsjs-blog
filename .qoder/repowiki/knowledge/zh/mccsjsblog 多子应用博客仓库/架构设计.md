根目录为纯工作区，不承载业务逻辑；各子模块以独立 npm 包形式存在并通过文件系统路径互相引用：
- backend_api 提供 Elysia + Better-Auth REST API，作为 frontend_public_site 与 admin_dashboard 的唯一数据源。
- shared_types 通过单一入口导出 Post/Author/Category/Tag/Comment 等实体接口，被 blog 与 admin 两个前端同时依赖，保证前后端领域模型一致。
- frontend_public_site（Astro）在构建期调用后端 API 生成静态页面；admin_dashboard（Vite+React）在运行时访问同一套 API 进行内容编辑。
- dev_scripts 提供备份恢复与 Hexo link.yml → Prisma 导入工具，面向运维与迁移场景。
- 根 package.json 仅声明 chokidar 作为全局 devDependency，无 workspaces 或 monorepo 编排，实际由 README.md 中的 cd 命令分别启动各子应用。
