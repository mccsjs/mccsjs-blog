本仓库采用按子应用独立 .env 加框架原生加载的轻量配置方案，未引入集中式配置中心或第三方配置库。各子应用的运行参数通过环境变量注入，并在各自入口文件中直接读取。

后端（Elysia + Prisma）
- 配置文件：backend/.env、backend/.env.example
- 数据库连接：Prisma 通过 datasource db { url = env("DATABASE_URL") } 从环境变量读取 SQLite 路径；backend/src/db.ts 仅暴露单例 PrismaClient，不处理任何配置逻辑。
- 服务启动参数：backend/src/index.ts 中 app.listen(process.env.PORT || 4000)，端口由 .env 的 PORT 控制。
- CORS 白名单：根据 NODE_ENV === 'production' 分支，从 FRONTEND_URL、ADMIN_URL 动态组装允许的 origin 列表，开发环境放行所有 localhost。
- 外部 API 密钥：友链测速接口使用 XXAPI_TOKEN（取自 process.env.XXAPI_TOKEN），默认空字符串时降级为直连探测。
- 上传目录：UPLOAD_DIR 在 .env.example 中声明，但当前代码硬编码 uploads/ 前缀，尚未接入该变量。
- 站点设置（运行时可编辑）：除上述进程级配置外，博客站点元信息（siteTitle、motto、TwikooEnvId 等）通过 SiteSetting 表持久化，提供 /api/settings GET/PUT 接口，前端管理后台可在线修改并即时生效。

前台静态站（Astro）
- 构建期配置：frontend/astro.config.mjs 内联定义 site、集成项、Vite server.allowedHosts 等，均为编译期常量，无运行时环境变量注入。
- 运行时行为：Astro 产物为纯静态文件，无需在运行时读取 .env。

管理后台（React + Vite）
- 构建期配置：admin/vite.config.ts 仅包含插件与别名，无 import.meta.env 使用点。
- 运行时行为：当前源码未发现对 process.env 或 import.meta.env 的引用，所有 API 地址、域名等似乎以硬编码方式消费后端 /api/* 路由。

约定与规则
- 每个子应用维护独立的 .env，禁止跨子应用共享环境变量。
- 敏感值（如 BETTER_AUTH_SECRET、XXAPI_TOKEN）必须放入 .env，不得提交到版本库；.env.example 仅保留占位说明。
- 新增环境变量应在 .env.example 同步补充注释，并在对应子应用入口处显式读取，避免隐式依赖。
- 站点外观类配置走数据库 SiteSetting 表而非环境变量，以便管理员在线调整。
